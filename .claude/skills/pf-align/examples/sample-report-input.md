# Sample pf-compare Report (Input Format)

This is the expected input format that pf-align parses in Step 2. Reports are generated by `/pf-compare` and stored at `docs/pf-compare/<component>.md`.

```markdown
---
component: badge
date: 2026-05-28
pf_version: Release 6.1
pf_url: https://www.patternfly.org/components/badge/html
pfj_url: http://localhost:1234/components/badge
completeness:
  pf_total: 3
  pfj_total: 2
  matched: 2
  missing_in_pfj:
    - with-tooltip
  extra_in_pfj: []
---

# PF Compare: badge

## Completeness

| # | PF Variation | PFJ Snippet | Status |
|---|---|---|---|
| 1 | Read | Read badges | matched |
| 2 | Unread | Unread badges | matched |
| 3 | With tooltip | --- | missing_in_pfj |

## DOM Comparison

### Read

**Status:** ok

### Unread

**Status:** differences_found

#### Missing CSS Classes
- `.pf-m-overflow` on `<span>` -- present in PF, absent in PFJ

## Action Items

1. **Add variation:** With tooltip -- implement badge with tooltip on hover
2. **Fix CSS:** Unread -- add `.pf-m-overflow` modifier class
```

pf-align extracts from this report:
- The `pf_url` from YAML frontmatter (used to open PatternFly docs for HTML extraction)
- The `## Action Items` section (parsed into typed items: `add_variation`, `fix_css`, etc.)
- Each item's number, type, title, and description
