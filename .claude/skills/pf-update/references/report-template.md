# Report Template

Use this template when writing the markdown report to `docs/pf-update/<VERSION>.md`. A companion JSON file (`<VERSION>.json`) is written alongside this markdown report using the schema from `report-schema.json`.

## YAML Frontmatter

```yaml
---
title: PatternFly Update Report
version: <VERSION>
date: <YYYY-MM-DD>
pfj_version: <PFJ_VERSION>
source_repos:
  - patternfly/patternfly
  - patternfly/patternfly-react
stats:
  total_prs: <TOTAL>
  relevant_prs: <RELEVANT>
  skipped_prs: <SKIPPED>
  breaking: <COUNT>
  features: <COUNT>
  fixes: <COUNT>
  new_components: <COUNT>
  token_changes: <COUNT>
  icon_changes: <COUNT>
components_affected:
  <LIST of component names, one per line with - prefix>
new_components:
  <LIST of new component names, one per line with - prefix, or empty []>
issues_created: <COUNT or 0>
---
```

## Body

```markdown
# PF Update: <VERSION>

**Date:** <YYYY-MM-DD>
**PFJ Version:** <PFJ_VERSION>
**PF Version:** <VERSION>

## Release Digest

- **Total PRs:** <TOTAL> (<RELEVANT> relevant, <SKIPPED> skipped)
- **Components affected:** <COMP_COUNT> (<comma-separated list>)
- **New components:** <NEW_COUNT> (<comma-separated list, or "none">)
- **Token changes:** <TOKEN_COUNT> PR(s)
- **Icon changes:** <ICON_COUNT> PR(s)

## Work Plan

| # | Pri | Component | Change | Source | Action |
|---|-----|-----------|--------|--------|--------|
<WORK_ITEMS>

## Unclassified PRs

<UNCLASSIFIED_LIST or "None">

## GitHub Issues

<ISSUE_LINKS or "No issues created">

---
*Generated by `/pf-update <VERSION>`*
```

## Placeholders

| Placeholder | Source |
|-------------|--------|
| `<VERSION>` | Target PF version from arguments |
| `<PFJ_VERSION>` | `version.patternfly` from `pom.xml` |
| `<YYYY-MM-DD>` | Current date |
| `<TOTAL>` | Total PR count from both repos |
| `<RELEVANT>` | PRs that passed filtering |
| `<SKIPPED>` | PRs discarded as noise |
| `<COUNT>` | Count for the specific category |
| `<COMP_COUNT>` | Number of existing PFJ components affected |
| `<NEW_COUNT>` | Number of new components detected |
| `<TOKEN_COUNT>` | Number of token-related PRs |
| `<ICON_COUNT>` | Number of icon-related PRs |
| `<WORK_ITEMS>` | Generated work plan rows (one `| ... |` row per item) |
| `<UNCLASSIFIED_LIST>` | Bullet list of unclassified PRs with links |
| `<ISSUE_LINKS>` | Bullet list of created GitHub issue links |
