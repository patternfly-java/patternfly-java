# GitHub Issue Templates

## Component update issue

Use for P1 (breaking), P2 (feature), and P3 (fix) work items on existing PFJ components.

**Title format:** `[PF <VERSION>] <component>: <summary>`

**Labels:** `pf-update`, `p<N>` (e.g., `p1`, `p2`, `p3`)

**Body template:**

```markdown
## PatternFly <VERSION> Update

**Priority:** P<N> — <RATIONALE>
**Component:** <COMPONENT>

### Changes

<LIST each PR as a bullet:>
- <PR title> ([#<number>](<url>))

### Suggested Actions

1. Run `/pf-compare <COMPONENT>` to identify gaps
2. Review the compare report at `reports/pf-compare/<COMPONENT>.md`
3. Run `/pf-align <COMPONENT>` to implement changes

### Source

- [PatternFly <VERSION> release](https://github.com/patternfly/patternfly/releases/tag/v<VERSION>)
- [PatternFly React <VERSION> release](https://github.com/patternfly/patternfly-react/releases/tag/v<VERSION>)

---
*Generated by `/pf-update <VERSION>`*
```

### Priority rationale text

| Priority | Rationale text |
|----------|---------------|
| P1 | Breaking change — PFJ HTML/CSS may already be incorrect |
| P2 | New feature — new variation or modifier to implement |
| P3 | Bug fix — PFJ may have the same issue |

## New component issue

Use for P4 work items (components not yet in PFJ).

**Title format:** `[PF <VERSION>] New component: <name>`

**Labels:** `pf-update`, `new-component`

**Body template:**

```markdown
## New PatternFly Component: <NAME>

A new component `<NAME>` was added in PatternFly <VERSION>.

### Source PRs

<LIST each PR as a bullet:>
- <PR title> ([#<number>](<url>))

### References

- [PatternFly docs](https://www.patternfly.org/components/<SLUG>)
- [PatternFly HTML source](https://github.com/patternfly/patternfly/tree/main/src/patternfly/components/<Name>)

### Decision

Evaluate whether to implement this component in PatternFly Java.
Consider: Is this component needed by downstream projects? Does it have a stable HTML/CSS implementation?

---
*Generated by `/pf-update <VERSION>`*
```

## Deduplication

Before creating an issue, search for existing open issues:

```bash
gh issue list --label pf-update --search "<component>" --state open --json number,title
```

If a matching issue exists (same component in the title), add a comment to the existing issue instead of creating a new one:

```bash
gh issue comment <number> --body "<comment body>"
```

The comment should list the new PRs found in the latest release, following the same "Changes" format from the component update template.
